# 火箭量化整合版 v2.6

## 四种报告体系

### 1️⃣ 个股分析报告（格式不动）
- **定位**：单只股票的深度量化分析
- **触发关键词**：分析股票、分析个股、分析股票代码
- **输出格式**：保持现有的4段式报告格式完全不变！
- **内容**：
  - 第1/4部分：股票基本信息 + 最新行情
  - 第2/4部分：技术指标分析（均线、MACD、RSI）
  - 第3/4部分：策略信号分析 + 策略回测结果
  - 第4/4部分：综合诊断 + AI分析（巴菲特视角、量化验证、投研分歧、操作建议）
- **必须加入AI分析**：所有报告都要加入AI分析，不能只有实际数据

- **触发方式**：定时任务自动触发，也可手动调用

  🧠 AI投资分析
  ----------------------------------------------------------------------------------------------------
  【扫描总结】
     本次扫描共筛选出3只优质股票，平均综合评分XX.X分，平均涨幅X.XX%，整体选股质量良好。

  【个股亮点】

### 3️⃣ 收益跟踪（午盘 + 收盘 双定时）

#### 👉 午盘（精简实时监控）
- **定位**：盘中盯盘、及时纠偏
- **要求**：不用深度分析，只看数据异动，**合并为一个完整报告一次性输出**，不拆分
- **触发关键词**：午盘报告、午盘收益、查看午盘
- **核心信息**：
  - 当日整体：账户总盈亏、收益率、跑赢/跑跌沪深300/中证500/所属行业指数
  - 持仓明细：个股盈亏、涨跌幅度、仓位占比、浮盈浮亏TOP3/踩坑TOP3
  - 风控快照：单笔个股最大回撤、仓位集中度、异动标的（突然拉升/跳水）
  - 极简归因提示：肉眼看「是大盘带的、板块带的，还是个股单独动」
- **必须加入AI分析**：所有报告都要加入AI分析，不能只有实际数据

#### 👉 收盘（完整版收益基底）
- **定位**：收盘后完整收益数据存档
- **触发关键词**：收盘报告、收盘收益、查看收盘
- **核心信息**：
  - 全口径收益：日收益、周累计、月累计、今年以来；基准对标收益差
  - 收益拆解：总收益拆分→现金仓位 + 持仓个股 + ETF/行业底仓贡献
  - 量化基础数据：波动率、当日夏普简易测算、盈亏比、换手影响（**只拉取持仓股票的历史数据，不要全量拉取**）
- **必须加入AI分析**：所有报告都要加入AI分析，不能只有实际数据

### 3️⃣ 投资报告（仅收盘 定时）
- **定位**：深度归因 + 资讯结合
- **触发关键词**：投资报告、今日投资报告
- **触发时机**：仅收盘后生成，每天一次
- **输出规则**：**合并成一个完整报告一次性输出**，保留4个部分内容结构不变，用分隔符区分，不拆分发送
- **核心输出**：
  - **专业收益归因**（alpha/beta）：
    - Beta 贡献：大盘/所属行业板块整体涨跌，给账户带来的躺赚收益
    - Alpha 贡献：持仓个股（中伟新材/赛腾股份/恒帅等）独立行情超额收益
    - 归因结论：今日赚钱靠大势？靠选股？靠个股独家利好？
  - **持仓关联财经资讯**（精准匹配，不堆垃圾新闻）：
    - 自动抓取：持仓个股公告、行业政策、产业链利好/利空、龙虎榜、机构调研
    - 强关联解读：哪条新闻→对应哪只票→影响今日涨跌（把行情和新闻钉死）
  - **复盘 + 次日极简策略提示**：
    - 今日亮点/隐患（比如：行业beta乏力，全靠个股alpha撑盘，持续性存疑）
    - 仓位优化建议、异动标的关注提醒、风控预警
  - **🧠 量化专家深度AI分析**：
    - 【收益质量深度量化评估】：动态分析alpha能力、beta对冲效果、胜率含金量等量化指标
    - 【持仓风险量化诊断】：动态分析集中度风险、板块分散度、流动性风险等风控指标
    - 【市场环境适配性分析】：动态分析当前策略与市场环境的匹配度，验证策略有效性
    - 【量化策略优化建议】：给出具体的仓位调整、止盈止损阈值、后续调仓信号等可执行建议
- **必须加入AI分析**：所有报告都要加入AI分析，不能只有实际数据

## 📁 历史数据优化方案

### 重要原则：只拉取持仓股票的历史数据，不要全量拉取！

1. **获取持仓列表** - 先看有哪些持仓股票
2. **只拉取持仓股票的历史数据** - 精准拉取，分析哪只拉取哪只数据
3. **避免全量拉取所有股票** - 节省时间、节省资源、速度大幅提升

### 具体实现
- 在 `return_tracker.py` 中，新增 `_calculate_quant_metrics` 方法
- 只获取持仓股票的历史数据（最近30-60天）
- 计算波动率、夏普比率、盈亏比等指标
- 保存每日收益数据到数据库，用于后续计算
